# Vibe Tutor Digest Rubric

A good digest is:

- Grounded in the current git diff, not a generic tutorial.
- Short by default: under 150 words unless the user asks for detail.
- Actionable: names changed files and changed functions/classes/routes when visible.
- Low hype: no motivational filler or broad claims about vibe coding.
- Honest: no correctness claim without tests, lint, build, or deterministic proof.
- Focused: at most 3 concepts from the actual change.
- Interactive: exactly one concrete self-check question.
- Verified: includes one clear verification status statement.
- Compact: no large code blocks; at most one excerpt, up to 12 lines, only if necessary.
